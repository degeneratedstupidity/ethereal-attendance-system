## Background

- **Backend**: FastAPI, SQLAlchemy 2.0 async, Pydantic v2. Routers for `attendance` and `users`. Services for attendance logic and course/enrollment management. JWT config keys already in `settings` but **no auth endpoints exist yet**.

- **Dependencies**: `python-jose`, `passlib[bcrypt]`, `bcrypt` are **already in `requirements.txt`** — auth can be implemented immediately. `python-multipart` needs to be added for file uploads.

### Phase 3 — Advanced Analytics (Bunk Calculator)

---

#### [MODIFY] `app/schemas/attendance.py`
Add new response schema `BunkCalculatorResult`:
```python
class BunkCalculatorResult(BaseModel):
    student_id: UUID
    student_name: str
    overall_percentage: float
    total_classes: int
    attended_classes: int
    threshold: float  # 75.0
    status: str  # "safe" | "critical" | "recovered"
    safe_to_bunk: int | None   # how many more they can miss and stay ≥75%
    must_attend: int | None    # how many consecutive they must attend to reach 75%
    message: str               # human-readable explanation
```

#### [MODIFY] `app/services/attendance_service.py`
Add new function `calculate_bunk_info(db, student_id)`:
```
Given: attended A, total T, threshold P = 75%
If current % ≥ P:
    Find max X such that (A / (T+X)) ≥ P
    → X = floor((A - P*T) / P)   [safe to bunk]
If current % < P:
    Find min N such that ((A+N) / (T+N)) ≥ P
    → N = ceil((P*T - A) / (1 - P))  [must attend]
```

#### [MODIFY] `app/routers/attendance.py`
Add new endpoint:
- `GET /api/v1/students/{student_id}/bunk-calculator` — student-accessible (own only), teacher-accessible (any)
